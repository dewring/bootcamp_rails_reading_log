# W7D2 — Open Library API Integration

## What fields does the search API return?
- `key` — the unique Book identifier, ex `/works/OL82563W`
- `title` — the book's title as a string
- `author_name` — an array of author names, ex `["J. K. Rowling"]`, because a book can have multiple authors
- `cover_i` — a number cover image ID used to build a cover URL
- `first_publish_year` — the year the book was first published

## What three fields do we store on Book and why?
- `ol_work_key` — uniquely identifies for that we never create duplicates
- `title` — saved to display the book to the user
- `author` — saved to display who wrote the book; we take `author_name.first` for multiple authors

## What are fetch_work and fetch_editions for? (Day 3 preview)
- `fetch_work` — I will use the ol_work_key to call OpenLibrary again and ask for the story description and details of the book.
- `fetch_editions` — I will call OpenLibrary to ask for the physical versions of the book (hardcover, paperback, specific publishers).
